## 1. 프로젝트 개요

| 항목 | 내용 |
|---|---|
| 가제 | Maze Defenders (변경 가능) |
| 장르 | 타워 디펜스 + 덱빌딩 + 미로 빌딩 |
| 플랫폼 | PC (Windows, Standalone) |
| 시점 | **2.5D 탑다운 (3D 에셋, 위에서 내려다보는 카메라)** |
| 엔진 | **Unity 6.3 (URP)** |
| 개발 인원 | **1인 (클라이언트)** |
| 개발 기간 | **3주** |
| 1판 플레이 타임 | 15~25분 |

**한 줄 컨셉**: 시작점→끝점 3D 타일 그리드 위에서, 플레이어가 벽을 세워 미로를 만들고, 그 벽 위에 카드로 얻은 유닛을 배치(= 타워)해 몰려오는 적을 막아낸다. **경로 자체를 플레이어가 설계**한다는 점이 기존 TD와의 차별점.

## 2. 작업 시 반드시 지킬 규칙

- **답변은 한국어로 한다.**
- **`Assets/Imported/` 폴더는 절대 건드리지 않는다.** (외부 에셋 영역)
- 1인 개발이므로, 코드 구조는 과도한 추상화보다 **3주 안에 굴러가는 단순한 구조**를 우선한다.
- 1주차 안에 "코어 루프 한 사이클이 완성된 상태"를 만든다 — 나머지는 그 위에 데이터로 쌓는다.

## 3. 게임 루프 요약

### 매크로 루프 (한 판)
```
게임 시작 → 맵 생성(벽 랜덤) → 기본 덱/자원 지급
  → [라운드 반복] 빌드 페이즈 → 전투 페이즈 → 보상
  → 모든 라운드 클리어 / 끝점 함락 → 결과
```

### 마이크로 루프 (1라운드)
```
라운드 시작: 마나 2 회복, 손패 5장 드로우
[빌드 페이즈]
  자원 카드 → 골드/샤드 / 효과 카드(마나) / 마법 카드(1회용)
  유닛 패널에서 벽 위로 드래그 → 타워 배치 (손패가 아닌 별도 패널)
  골드로 벽 생성/제거, 상점 카드 구매(골드+마나1), 샤드 교환
  막힘 체크 통과해야 전투 진입
[전투 페이즈]
  웨이브 진행, 타워/유닛 자동 공격, 마법 사용 가능
  보스 처치 시 보스 보상
라운드 종료: 손패/사용 카드 전부 묘지로
```

## 4. 핵심 시스템

### 4.1 맵
- **20 × 10 그리드**, XZ 평면(Y는 높이축)
- **Unity Tilemap을 쓰지 않는다.** 외부 에셋에 맞는 2D 타일이 없어서, 씬의 `MapTiles` GameObject 아래에 **3D 메시 타일(`SM_Env_Pillar_Square_Base_01` 등)을 직접 배치**한 것을 그리드로 사용 (총 200개)
- 셀 크기 ≈ **3.48 × 3.48 유닛** (XZ). 그리드 ↔ 월드 좌표 변환은 `MapTiles`의 자식 타일 위치에서 역산
- 시작점 = 왼쪽 위 (Top-Left), 끝점 = 오른쪽 아래 (Bottom-Right). **매판 고정**
- 게임 시작 시 1회만 벽 랜덤 생성. 시작점/끝점 인접 타일 중 하나는 반드시 열려 있어야 하고, 시작→끝 경로가 1개 이상 보장돼야 함
- **A\* 길찾기는 벽이 변할 때만 재계산.** 빌드 페이즈에서 플레이어가 벽 설치/제거할 때만 트리거 — 전투 페이즈에는 벽이 변하지 않음

### 4.2 벽
- 빈 타일에 골드로 설치, 골드로 제거
- **준비(빌드) 페이즈에서만 설치/제거 가능** — 전투 페이즈 중 불가
- **적은 벽을 파괴하지 않는다.** 오직 플레이어만 벽을 변경
- 벽 제거 환불 규칙:
  - 설치한 그 라운드 안에 제거 → **100% 환불**
  - 다음 라운드 이후 제거 → **환불 없음 + 철거 비용 추가 지불**
- 빌드 페이즈 종료 시 경로가 막혀 있으면 전투 진입 불가, 막힘 구간 하이라이트

### 4.3 타워 = 유닛 (유닛 패널 기반)
**핵심 원칙**: 별도의 "타워 건설" 단계는 없다. **유닛 카드는 획득 시 유닛 패널(인벤토리)에 등록되고, 그 패널에서 벽 위로 드래그&드롭하면 그 자체가 타워**가 된다. 손패에서 직접 배치하는 방식이 **아님**.

- **획득 흐름**: 상점 구매·보스 보상·랜덤 박스 등으로 유닛 카드를 얻으면 → **유닛 패널에 등록** + 덱에도 추가됨 (두 영역에 동시에 존재)
- **배치 방식**: 빌드 페이즈에 **유닛 패널에서** 벽 타일 위로 드래그&드롭. 손패와 무관
- 벽 타일 위에만 배치 가능 (빈 벽은 단순 장애물)
- 한 벽 타일당 유닛 1명 (다중 슬롯 X). 다른 유닛 드롭 시 교체
- 타워 스탯은 등록된 유닛 카드의 스탯 그대로
- **카드 1장 = 유닛 인스턴스 1개**의 1:1 대응. 같은 카드를 두 번 획득하면 패널에 두 인스턴스로 등록되어 각각 다른 벽에 배치 가능

### 4.4 카드 종류 (4종)

| 종류 | 사용 방식 | 마나 비용 |
|---|---|---|
| **유닛** | 획득 시 **유닛 패널에 등록** → 패널에서 벽 위로 드래그&드롭하여 타워 배치. **손패에 잡혀도 일반 유닛 카드는 사용 불가 = 방해 카드** (자원/효과 슬롯만 차지). **유닛/효과 이중 카드**일 경우에만 효과로 사용 가능 → 사용 시 묘지로 | 배치엔 X / 이중 카드 효과 사용 시 해당 비용 |
| **효과** | 손패에서 사용 → 즉시 효과 (드로우, 카드 파괴, 마나 획득, 유닛 공격력 업, 광역 데미지, 슬로우, 상점 리롤, 매물 슬롯 확장 등) | 0~3 |
| **자원** | 손패에서 사용 → 골드/샤드 획득 | 0 |
| **마법** | 효과 카드로 획득 → **별도의 마법 탭(인벤토리)** 에 적립 → 1회용 사용 | **마나 사용 X** |

### 4.5 마법 카드 규칙
- 덱에 포함된 일반 카드가 **아님**. 별도 마법 탭에 누적
- **최대 5장 보유.** 가득 차면 신규 카드는 즉시 폐기 (기존 5장은 유지)
- 라운드 종료 시 미사용 마법은 **다음 라운드로 이월**
- 각 마법은 **"준비 페이즈 전용" 또는 "전투 페이즈 전용"** 고정 속성을 가짐 — UI에 아이콘 명시, 맞지 않는 페이즈에서는 비활성화
  - 준비용: 자원/드로우/이번라운드 강화
  - 전투용: 방해(슬로우/스턴), 공격(단일/광역), 단시간 버프, 체력 % 깎기
- **체력 % 비율 감소형 마법은 보스에게 적용되지 않음** (즉사 콤보 방지)

### 4.6 카드 텍스트 컨벤션 ("업그레이드" 지속 시간)

| 표기 | 적용 |
|---|---|
| "공격력 업그레이드" (수식어 없음) | **해당 판 영구** |
| "이번 라운드 공격력 업그레이드" | 해당 라운드 1회 한정 |
| "일시적 공격력 업그레이드" | 카드/효과 명시 시간(예: n초) |

> 컨벤션이 무너지면 플레이어가 강화 지속을 오해하므로, 보스 보상/영구 강화는 수식어 없음, 준비 페이즈 마법은 "이번 라운드", 전투 단시간 버프는 "일시적" 접두어를 반드시 사용.

### 4.7 마나
- 라운드 시작 시 **2 회복** (라운드 진행과 무관하게 고정)
- 효과 카드 사용 시 마나 비용만큼 소모
- **상점 카드 구매 시 골드 + 마나 1 고정 추가 소모** (가격 무관)
- 라운드 종료 시 잔여 마나 **소멸 (누적 X)**
- 후반 라운드 자동 마나 증가 **없음** — 확장은 카드(효과/마법)로만
- 마나 비용 분포 (권장): 0마나 30% / 1마나 40% / 2마나 20% / 3마나 10%

### 4.8 드로우 / 묘지 / 덱
- 매 라운드 시작 시 **기본 5장** 드로우. 효과 카드로 매수 증감 가능
- **손패 상한 20장.** 초과 드로우는 상한까지만 채우고 나머지는 **덱에 그대로** 남음 (묘지로 X)
- **덱이 비면 드로우 불가** — 라운드 도중 묘지→덱 복귀 없음
- 사용한 카드는 즉시 묘지로. 같은 라운드 안에서 같은 카드 재사용 불가
- 라운드 종료 시 손패 잔여 카드 전부 묘지로
- **다음 준비 페이즈 시작 시 묘지→덱 복귀 후 셔플** → 손패 드로우. 묘지는 라운드 단위로 비워지는 임시 영역

**유닛 카드의 특수 운용**
- 유닛 카드는 획득 시 **유닛 패널에 등록 + 덱에도 포함** (두 영역에 동시 존재). 타워로 배치되어도 덱에서 사라지지 않음 — 매 라운드 손패로 들어올 수 있음
- 손패에 잡힌 유닛 카드는 **사용 불가 = 방해 카드** (효과/자원 슬롯만 잡아먹음). 라운드 종료 시 다른 카드와 같이 묘지로 → 다음 라운드 셔플 풀로 복귀
- 예외: **유닛/효과 이중 카드**는 손패에서 효과로 사용 가능 → 사용 시 묘지로
- 손패에 잡혀도 패널 등록과 배치된 타워는 영향 없음. 두 흐름은 분리됨

**카드 파괴 효과**
- 묘지가 아닌 **완전 삭제** (셔플 풀에서 영구 제거)
- **유닛 카드 파괴 시**: 패널 등록도 제거되며, 해당 카드 인스턴스가 배치된 타워 유닛도 맵에서 함께 사라짐. **같은 종류(동일 카드명)의 다른 인스턴스는 영향 X** — 카드 1장 = 유닛 인스턴스 1개의 1:1 대응이기 때문

### 4.9 상점
- 게임 시작 시 매물 **초기 6장** 진열 (기본 슬롯 6)
- **준비 페이즈에서만** 구매 가능
- 가격은 등급별 **기본가 고정**, 효과/특수 효과로 증감 가능
- 구매 시 **골드 + 마나 1** 소모 (마나 0이면 구매 불가)
- **매물 자동 갱신 없음** — 라운드 종료 시 미구매 매물은 그대로 유지
- 6장 모두 구매하면 **품절** 상태. 자동으로 채워지지 않음
- 새 매물을 받으려면:
  - 효과 카드의 **상점 리롤** (디자인 예정) — 사용 시 매물 6장 재추첨. **기존에 진열된 카드는 후보 풀에서 제외**
  - **매물 슬롯 확장 효과 카드** — 슬롯 상한 6 → +증가 (해당 판 누적)
  - 보스 보상의 리롤권, 일부 마법 카드
- **골드로 리롤 불가**. **무료 카드 드래프트 보상 없음** — 덱 확장은 오직 상점 구매로만

### 4.10 자원

**마나**: 카드 사용 자원. 라운드 시작 2 회복, 종료 시 소멸.

**골드**: 구매 자원. **라운드 종료 시 0으로 초기화** (단기 자원).
- 획득: 자원 카드, 적/보스 처치 보상, 샤드1 교환
- 소비: 벽 생성/제거, 상점 카드 구매, (선택) 카드 제거

**샤드**: 보조 자원. **라운드 간 누적 유지** (장기 비축 가능).
- 획득: 자원 카드(주된 출처), 일부 효과 카드, 보스 보상
- 교환만 가능: **샤드1 → 골드 1**, **샤드2 → 마나 1**
- 사실상 라운드 간 가치 저장 수단 (골드는 라운드 종료 시 사라지므로)

### 4.11 준비 페이즈 타이머
- 초기 (1라운드): **600초**
- 라운드 진행에 따라 **+50초씩 증가**
- 상한 **800초** (5라운드부터 상한 도달)
- 만료 시 강제 전투 진입. 단 경로 막힘 시 진입 불가, 플레이어 대기
- 준비 페이즈에서만 진행, 전투 페이즈는 정지. 잔여 시간은 다음 라운드로 이월 X

### 4.12 적
- A*로 시작점 → 끝점 이동
- 끝점 도달 시: 일반 적 **체력 -1**, 보스 **체력 -10**
- 플레이어 시작 체력: **20** (보스 2회 통과 시 사망)
- 적은 벽을 파괴하지 않음
- MVP 적 종류: **기본 / 고속 / 탱크 + 보스 1종** (벽 파괴자·특수형 제외)
- 웨이브 데이터는 ScriptableObject 관리

### 4.13 라운드 간 초기화 정리

| 항목 | 다음 라운드 시작 |
|---|---|
| 골드 | **0으로 초기화** |
| 마나 | **2로 회복** (잔여 소멸) |
| 샤드 | **유지** |
| 손패 | **5장** (효과로 증감, 상한 20) |
| 묘지 | 전부 덱으로 복귀 후 셔플 |
| 상점 매물 | **자동 갱신 X** (미구매 매물 그대로) |
| 타이머 | 이전 + 50초 (상한 800) |
| 마법 탭 | 미사용분 유지 (상한 5) |

### 4.14 보스 보상 풀 (5/10라운드)
- 다음 라운드 추가 손패 (1라운드 한정, 예: +2장)
- 랜덤 박스 (카드 1종 영구 획득, 덱 추가)
- 유닛 공격력 업그레이드 (해당 판 영구)
- 상점 리롤권 (n회)
- 자원 보너스 (골드/샤드)

## 5. MVP 콘텐츠 스코프

## 6. 기술 스택 & 아키텍처

| 분야 | 선택 |
|---|---|
| 엔진 | Unity 6.3, URP |
| 시점 | **2.5D 탑다운 (3D 에셋, Orthographic 또는 살짝 기울인 탑다운 카메라)** |
| 그리드 | **씬에 직접 배치한 3D 메시 타일** (`MapTiles` 하위). Unity Tilemap 미사용 — 외부 에셋에 맞는 2D 타일이 없어 코드 생성 포기, 직접 배치 데이터를 그대로 사용 |
| 길찾기 | A* (직접 구현 또는 A* Pathfinding Project 에셋), 노드 = 직접 배치한 타일 좌표 |
| 데이터 | ScriptableObject (카드/유닛/적/웨이브) — **타워는 별도 데이터 없음(유닛=타워)** |
| UI | UI Toolkit 또는 uGUI |
| 애니메이션 | DOTween |
| 입력 | Unity InputSystem |
| 사운드 | AudioSource + 무료 SFX/BGM |
| 폰트 | TextMeshPro |

### 아키텍처 핵심 원칙
- **카드 효과 일반화**: `ICardEffect` 인터페이스 + 데이터 + 효과 리스트 조합. 새 효과 추가 = 클래스 한 개만 작성
- **이벤트 기반 결합**: `OnWaveStart`, `OnEnemyKilled`, `OnUnitPlaced`, `OnWallPlaced` 등으로 시스템 간 결합 최소화
- **A\* 트리거**: 벽 추가/제거 시점에만 재계산. 매 프레임 X
- **오브젝트 풀링 필수**: 적, 투사체, 이펙트
